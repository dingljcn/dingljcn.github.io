PicGo能够让我们将博客中的图片都存储到远程仓库中集中管理, 也算一个比较重要的功能

二、安装

官方文档

2.1 安装hexo

2.1.1 hexo-cli

npm i -g hexo-cli: 该命令用于安装hexo脚手架

2.1.2 hexo init

hexo init: 该命令用于初始化博客结构

出现以上错误, 则重新通过管理员打开命令行, 执行set-executionpolicy remotesigned

然后重新执行hexo init

初始化成功, 得到以下目录结构

几个核心的目录结构:

  • scaffolds: 模板
  • source:
    • _posts: 文章, 你的所有博客markdown文档都放在这里
    • img: 图片, 你的所有图片都放在这里, 无论是_config.yml还是自己的博客中, 如果需要引用图片, 都是/img/${ 图片名称 }的形式
  • themes: 下载的主题会放到这里 主题商店
  • _config.yml: 全局配置

2.1.3 hexo s

hexo s: 该命令用于在本地启动服务

2.2.1 scaffolds/post.md

配置如下内容, 然后执行命令hexo new post 新文章名, 即可按照以下模板, 快速创建一篇新的博客文章了

2.2.2 _config.yml

_config配置说明

只需修改以下内容即可

2.3 部署hexo

2.3.1 创建仓库

仓库名必须按照用户名.github.io的规范

创建后回到标题 2.2.2 _config.yml配置url和deploy

  • 以后你的博客地址就是https://用户名.github.io, 例如: dingljcn.github.io

2.3.2 部署

npm install hexo-deployer-git --save: 第一次部署需要安装该插件

hexo clean: 该命令用于清除上次编译的结果

hexo deploy或hexo d: 这两个命令等价, 都是部署的意思

  • 如果你仓库名搞错了, 比如写成了dingljcn, 并且已经部署了一次, 现在重新创建了dingljcn.github.io, 想要重新部署, 那么首先你要修正_config.yml的内容, 其次你要删除根目录下的db.json和.deploy_git, 因为这里面缓存了很多数据, 可能会影响代码提交

三、更换主题(以butterfly为例)

主题商店

3.1 安装butterfly

  1. 在根目录下将代码clone下来

  2. 修改_config.yml修改theme的值为butterfly

  3. 安装渲染插件npm i hexo-renderer-pug hexo-renderer-stylus --save

  4. 启动hexo查看效果(hexo s)

3.2 配置butterfly

找到/theme/butterfly/_config.yml进行配置, 这里就不作说明了: 教程 | Butterfly

注意, 如果想要启用某些功能, 那么对应的插件必须要安装:

  • npm i --save hexo-wordcount: 字数统计

3.3 注入css/js

在根目录下创建如下两个文件

  • /source/js/dingljcn.js

  • /source/css/dingljcn.css

然后修改butterfly下的_config.yml, 注意不要找到根目录的_config.yml去了
